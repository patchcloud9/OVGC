Managing Events

This guide covers what to do after an event is created: editing details, cancelling occurrences, restoring cancelled events, and permanently deleting events. See the Events guide for creating new events.

Editing an Event

https://www.okanoganvalleygolf.com/admin/events/{id}/edit

Click Edit on any event in the admin list to open the edit form. All fields from the create form are available.

Editing a recurring series changes ALL occurrences. If you change the title, time, or description on a recurring event, every past and future occurrence is updated immediately. There is no way to change only one occurrence of a recurring series — use the Cancel a Single Occurrence approach for one-off changes.

Updating Blackout Dates on an Existing Event

The Blackout Dates section shows all currently saved skip dates as removable tags. To update them:

  1. Click the × on any tag to remove that blackout date.
  2. Use the date picker to add new blackout dates.
  3. Click Save Changes.

When you save, the system replaces all blackout dates with whatever is shown in the tags — removed tags are gone, new tags are saved.

Cancelling Events

Cancellations show the event on the public calendar with a Cancelled badge. Members can see the event was planned but called off. This is different from a blackout date, which removes the occurrence silently.

Cancel From a Date Forward

Use this when a recurring series is stopping — for example, a league that was supposed to run through August but is ending in June.

  1. In the admin event list, click Cancel next to the event.
  2. Enter the Cancel From Date — the first date you want cancelled (inclusive).
  3. Click Cancel Event.

All occurrences on or after that date will show as cancelled on the calendar. Occurrences before that date remain active and unaffected.

Cancel the Entire Event

If you set the Cancel From Date equal to the event's original start date, the entire series is cancelled — including all past and future occurrences. The event remains in the database and stays visible on the calendar (with a Cancelled badge) so members can see the history.

Cancelling does not delete the event. Use Delete only when you want to permanently erase all record of it.

Handling a One-Off Cancellation Within a Recurring Series

The cancel form applies a date range, not a single date. To cancel just one occurrence of an ongoing series (e.g., next Wednesday's league is rained out but resumes the week after):

  1. Add the date as a Blackout Date on the edit form — this removes it silently from the calendar.
  2. Or, if you want members to see it as "Cancelled that week", add a separate one-time event for that specific date with the same title and mark it cancelled. (This is uncommon — most clubs use blackout dates for silent skips and post a banner or news item for high-visibility cancellations.)

Restoring a Cancelled Event

If an event was cancelled by mistake, or the situation changed and the event is back on:

  1. In the admin event list, find the cancelled event (it will show a "Cancelled" status badge).
  2. Click Restore.
  3. Confirm the action.

This clears the cancellation and all occurrences become active again. Any blackout dates that were set previously are not affected — they remain in place.

Restoring removes the cancelled_from date entirely. If you had a partial cancellation (cancelled from a future date) and restore it, the entire series goes back to active — including the occurrences that were previously marked cancelled.

Deleting an Event Permanently

Deleting an event removes it and all associated data — every occurrence, every result, every exception — from the database. This cannot be undone.

  1. In the admin event list, click Delete next to the event.
  2. A confirmation checkbox will appear — check it to confirm you understand the deletion is permanent.
  3. Click Delete Event.

Permanent and unrecoverable. If the event has historical results posted (scores, leaderboards), deleting the event removes those results too. Consider cancelling instead of deleting if the event has history worth keeping.

Cancel vs. Delete — Which Should I Use?

SituationRecommended action
Season is over, event won't run next yearCancel from the last occurrence date. History stays visible.
Event was created by mistake, never ranDelete — no history to keep.
Event has posted results you want to preserveCancel — keeps results on the public detail page.
Duplicate event accidentally createdDelete the duplicate.
Event details need major correctionEdit — don't cancel and recreate.

Tips & Common Mistakes

Verify your change on the public calendar.
After editing, cancelling, or restoring an event, open /events in a new tab and navigate to the affected month to confirm the calendar looks correct.

Use Blackout Dates for weather/holiday skips.
For a recurring event that simply won't happen one week (holiday, weather), a blackout date is cleaner than a cancellation — it silently skips that week and members won't see any notice of it.

Editing changes every occurrence.
If you need to adjust only a future part of a series (e.g., the start time is changing from 8 AM to 9 AM starting next month), the cleanest approach is to cancel the old event from the changeover date and create a new event with the updated time starting from that same date.

Cancelled events still appear on the calendar.
This is intentional — members can see that the event was planned but won't be happening. If you want it gone entirely, delete it. If you want it gone from one specific date forward only, use a Cancel From Date on that occurrence.
