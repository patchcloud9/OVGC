Contact Page

The Contact page lets visitors send a message directly to the club. When the form is submitted, the message is emailed to the club's contact address — there is no inbox inside the admin panel. This guide covers what visitors see, how to configure where messages go, and how to adjust related settings.

What Visitors See

The public contact page at /contact has a two-column layout:

ColumnContents
LeftClub name, address, phone number, email address, and an embedded Google Map
RightThe contact form (Name, Phone, Email, Comment fields)

After a visitor submits the form, they see a success message on the same page. The message is immediately emailed to the club's contact address.

Screenshot suggestion: public /contact page showing both columns — club info on the left with the map, and the filled-in form on the right.

Where Messages Go

Submitted messages are emailed to the Contact Email address set in Theme Settings. To change where contact form submissions are delivered:

  1. Go to Admin → Theme Settings (/admin/theme).
  2. Find the Contact Email field.
  3. Enter the email address that should receive form submissions.
  4. Click Save Settings.

No contact email = no delivery. If the Contact Email field in Theme Settings is blank, the form will appear to work but messages will not be sent. Always verify a valid email is set before the site goes live.

The contact info displayed on the page also comes from Theme Settings. The club name, address, phone, and email shown in the left column are all pulled from Admin → Theme Settings. Update them there to keep the contact page current.

Contact Form Fields

FieldRequiredNotes
NameYesVisitor's name, included in the email subject and body.
Phone NumberNoOptional. Included in the email body if provided.
EmailYesMust be a valid email address. Set as the Reply-To on the outgoing email so you can reply directly.
CommentYesThe message body. No length limit.

If a visitor submits the form with missing or invalid fields, the page reloads with an error message and their input is preserved so they can correct only what needs fixing.

Spam Protection

The contact form has two layers of spam protection built in:

Rate Limiting

The form accepts a maximum of 5 submissions per 5 minutes from the same browser. If a visitor hits that limit, they see an error message and must wait before trying again. This is automatic — no configuration needed.

reCAPTCHA v3

If reCAPTCHA v3 keys are configured in config/config.php, the form silently scores each submission in the background. Submissions that score below the threshold (0.5) are rejected with an error message. Visitors never see a checkbox or image challenge — it runs invisibly.

reCAPTCHA is optional. If the keys are not configured, the form works normally without it. If the Google reCAPTCHA API is unreachable (network issue), the form fails open — the submission is allowed through rather than blocking a legitimate visitor.

SMTP / Email Configuration

The contact form sends email through the site's SMTP configuration. These settings live in config/config.php on the server and are set up once by whoever manages hosting. They are not editable from the admin panel.

SettingPurpose
MAIL_HOSTSMTP server hostname (e.g., mail.okanoganvalleygolf.com)
MAIL_PORT587 for STARTTLS, 465 for SSL
MAIL_USERNAMESMTP login username
MAIL_PASSWORDSMTP password or app password
MAIL_FROM_ADDRESSThe "From" address on outgoing emails
MAIL_FROM_NAMEThe display name shown in the From field
MAIL_ENCRYPTIONtls (default) or ssl

If contact form submissions are not arriving, first check that the Contact Email in Theme Settings is correct, then ask your hosting provider to verify the SMTP credentials in config/config.php. Also check your spam/junk folder — some mail servers flag automated messages.

Tips & Gotchas

Reply directly from your email client.
Each submission sets the visitor's email address as the Reply-To header. Open the notification email and click Reply — your response goes straight to the visitor, not back to the site.

There is no message archive in the admin panel.
Contact form submissions are emailed and not stored in the database. If you need a record of messages, keep them in your email inbox or set up a filter to tag them automatically.

The rate limit applies per browser, not per IP.
If you are testing the form yourself and hit the limit (5 submissions in 5 minutes), wait a few minutes before trying again. This is expected behavior — not a bug.

Contact info on the page is controlled by Theme Settings.
If the address, phone, or email shown on /contact needs updating, go to Admin → Theme Settings — not to any contact-specific admin page.
