Public Rates Page: /rates

The public rates page is the visitor-facing green fees and cart rental page. It displays active rate groups and each group's rates in sorted order.

  • Route: GET /rates
  • Controller: App\Controllers\RatesController::index()
  • View: app/Views/rates/index.php

Data source

Rate data comes from:

  • App\Models\RateGroup::allGroups(true) for active groups
  • App\Models\Rate::where(['group_id' => $g['id']]) for group rates
  • App\Models\RatePageContent::getContent() for editable content blocks (above/below sections, rules text)

Display logic

  • Groups sorted by group sort_order and active flag
  • Rates within groups sorted by sort_order
  • Rules text parsed into bullet list lines (non-empty newline-separated lines)

Admin Rates Dashboard: /admin/rates

Admin rates management consists of two layers:

  • Rate groups via RateGroupController
  • Rates within groups via RateController

Rate group routes

  • GET /admin/rates — list groups (controller RateGroupController::index, view app/Views/rates/admin.php)
  • GET /admin/rates/create — show create group form
  • POST /admin/rates — create group (CSRF + auth admin)
  • GET /admin/rates/{id}/edit — edit group form
  • PUT /admin/rates/{id} — update group
  • DELETE /admin/rates/{id} — delete group

Rate routes (per-group)

  • GET /admin/rates/{groupId}/rates — list rates in group
  • GET /admin/rates/{groupId}/rates/create — create rate form
  • POST /admin/rates/{groupId}/rates — store rate
  • GET /admin/rates/{groupId}/rates/{rateId}/edit — edit rate form
  • PUT /admin/rates/{groupId}/rates/{rateId} — update rate
  • DELETE /admin/rates/{groupId}/rates/{rateId} — delete rate

Rates page content editing

RateGroupController::updateContent() handles editing page text sections and optional scorecard upload:

  • POST /admin/rates/content (auth+role:admin+csrf)
  • Editable fields: rules_text, above_rates, below_rates
  • Optional scorecard image upload (jpg/png) saved to /public/assets/scorecard.{jpg|png}

Validation and security

  • `RateGroupController` validates slug/title/subtitle/note/sort_order/active
  • `RateController` validates description/price/sort_order/notes
  • All admin rate routes use `auth` + `role:admin` middleware
  • CSRF protected forms plus proper flash feedback on validation errors
